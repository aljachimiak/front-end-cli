What does front-end mean?

On the web, it's the things you see and interact with in a browser window.

Components of the front-end

Demo

A simple web page.

Back in the olden days...

  • "Hey I need a web page!"
  • Handed an Indesign or Photoshop comp
  • Open up DreamWeaver or FrontPage and recreate the comp...
  • ...with some sweet table-based layouts and inline styles

Still possible today

Macaw provides the same flexibility as your favorite image editor but also writes semantic HTML and remarkably succinct CSS.

A likely modern scenario:

  • "Hey I need a web platform!"
  • that is fast
  • looks great on all devices
  • has integrated analytics
  • and can be maintained by the marketing team

    • Content Management System

    • Drupal
    • WordPress

    Grab a theme and insert your content.

    Where does a front-end developer fit in in this scenario?

    Creating and/or Customizing the CMS theme

    Creating CMS themes

    Need to know:

    • html, js, and css
    • back-end language of the CMS (usually Php)

    Working on a web-app?

    Each back-end language deals with views differently

    Language Framework Views/Templates
    Ruby Rails Embeded Ruby (html.erb)
    Python Django Jinja2
    Php Laravel Blade
    Node Express HandleBars
    C# MVC .cshtml

    The burden of an additional language has been recognized...

    Drupal and WordPress now offer api's that offer the possibility of having a front-end that is decoupled from the back-end.

    The landscape is vast...

    Let's narrow the scope of front-end for this talk:

    • Only html, css, and js
    • producing the assets that go in the <head> and the <body>
    • tasks that make the web experience faster (ex: minification, concatatenation)
    • ...and the dev environment needed to produce all this stuff
